# Your Files

> Your content and app-level overrides live next to the NuDocs-specific folders.

## Common Project Files

- Numbered markdown folders such as `1.docs/` and `2.deploy/` define navigation order.
- Standalone markdown files such as `3.config.md` create additional top-level pages.
- `.docs/pages/` lets you add or replace Nuxt routes.
- `.docs/components/` lets you register or override Vue components used by the docs app.

## Rule Of Thumb

- Put markdown content in numbered folders or markdown files.
- Put app configuration and assets in `.docs/`.
- Put generated output in `.output/`.
