## Quick Start

Create a new `docs/` project:

```bash [npm]
npm create nudocs@latest docs
```

```bash [pnpm]
pnpm create nudocs@latest docs
```

```bash [yarn]
yarn create nudocs docs
```

```bash [bun]
bun create nudocs docs
```

Go to the docs dir `cd docs/`

Start development server:

:pm-run{script="dev"}

Build for production:

:pm-run{script="build"}

The generated build is written to `.output/` inside your docs project. For a standard setup, the full path is `docs/.output/`, and the directory most static hosts should publish is `docs/.output/public/`.

## Docs Structure

NuDocs keeps project-specific files under `.config/` and `.docs/`:

- `.config/docs.yaml` contains the main site configuration.
- `.config/automd.ts` is optional and can register custom `automd` generators.
- `.docs/public/` serves static assets.
- `.docs/pages/` is optional and lets you add or replace Nuxt routes in the docs app.
- `.docs/components/` is optional and auto-registers custom Vue components for the docs app.
- `1.docs/`, `2.deploy/`, `99.blog/`, and other numbered folders contain your content and control navigation order.
- Standalone files such as `3.config.md` create top-level pages without needing a folder.

Build artifacts are generated in `.output/`, which is separate from `.docs/` and should not be edited manually.

## Before Production Builds

Set `url` in `.config/docs.yaml` before publishing:

```yaml [.config/docs.yaml]
url: "https://example.com"
```

This is required for production SEO metadata and generated files such as `/llms.txt`.

See [Customizations](/docs/customization/overview) for route and component overrides, and [Deploy](/deploy) for publishing `docs/.output/public/`.
