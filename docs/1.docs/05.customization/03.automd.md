# automd

> Extend markdown generation with custom `automd` generators.

When `automd: true` is enabled in `.config/docs.yaml`, NuDocs transforms markdown files containing `<!-- automd:... -->` blocks while content is parsed.

## Built-In Generators

Built-in generators such as `badges`, `file`, and `jsdocs` work without extra setup:

```md
<!-- automd:badges color="yellow" name="defu" -->

[![npm version](https://img.shields.io/npm/v/defu?color=yellow)](https://npmjs.com/package/defu)
[![npm downloads](https://img.shields.io/npm/dm/defu?color=yellow)](https://npm.chart.dev/defu)

<!-- /automd -->
```

## Custom Generators

Register custom generators in `.config/automd.ts`:

```ts [.config/automd.ts]
import { defineGenerator, type Config } from "automd";

export default <Config>{
  generators: {
    test: defineGenerator({
      name: "test",
      async generate() {
        return {
          contents: "automd works!",
        };
      },
    }),
  },
};
```

Then call the generator with the matching block name:

```md
<!-- automd:test -->

automd works!

<!-- /automd -->
```

`test` is a custom generator here, so it only runs for `automd:test`. It does not change how built-in generators such as `automd:badges` behave.
