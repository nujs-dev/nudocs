## Overview

NuDocs layers your docs project on top of the bundled app.

Three customization points matter most:

- `.docs/pages/` is mounted as a Nuxt pages layer.
- `.docs/components/` is auto-registered with high priority.
- `.config/automd.ts` lets you register custom `automd` generators.

This lets you add new routes and components, replace built-in ones, and extend generated markdown content while keeping the bundled app intact.

## What Gets Overridden

- A file in `.docs/pages/` overrides the built-in page component for the same route.
- A file in `.docs/components/` overrides the built-in component with the same name.
- A generator in `.config/automd.ts` only runs when the `automd:` block name matches its key.

## Notes

- Keep overrides as small as possible so theme updates remain easy to absorb.
- When replacing built-in components, preserve expected props and slots if other parts of the app rely on them.
- If you only need to author content, stay in markdown files and ignore these hooks.
