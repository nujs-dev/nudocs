# Pages

> Extend or replace routes from `docs/.docs/pages/`.

NuDocs mounts `.docs/` as a Nuxt layer, so files inside `.docs/pages/` use standard Nuxt file-based routing.

## Add Routes

Use `.docs/pages/` when you want extra application routes that are not backed by markdown content:

```text
docs/
  .docs/
    pages/
      playground.vue
```

This adds a `/playground` route.

## Override Built-In Routes

Use the same file path as a bundled page to replace it:

```text
docs/
  .docs/
    pages/
      index.vue
      blog/
        index.vue
```

In this setup:

- `.docs/pages/index.vue` replaces the default landing page.
- `.docs/pages/blog/index.vue` replaces the default blog index page.

## Known Built-In Routes

- `.docs/pages/index.vue`
- `.docs/pages/[...slug].vue`
- `.docs/pages/blog/index.vue`
- `.docs/pages/blog/[...slug].vue`

If you override a built-in route, you are taking full ownership of that page component.
