# Components

NuDocsNuDocs> Add app-level Vue components or replace built-in ones from `docs/.docs/components/`.

NuDocs mounts `.docs/` as a Nuxt layer, so components in `.docs/components/` are auto-registered before the bundled theme components.

## Add New Components

Create any Vue component inside `.docs/components/` and use it directly from markdown or custom pages:

```text
docs/
  .docs/
    components/
      HeroBanner.vue
```

```mdc
<HeroBanner />
```

## Override Built-In Components

Use the same component name as the built-in implementation when you want to replace part of the theme:

```text
docs/
  .docs/
    components/
      AppHeader.vue
```

This makes `AppHeader.vue` from your docs project take precedence over the default header component.

## Tips

- Keep the public props and slots compatible when replacing a built-in component.
- Put one-off page UI in `.docs/pages/` if it is only used by a single custom route.
- Put reusable UI in `.docs/components/` if it should be available across markdown and Vue pages.
