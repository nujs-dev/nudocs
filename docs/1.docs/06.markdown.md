# Markdown

> Write docs with standard Markdown, MDC components, and `automd` blocks.

NuDocs is designed so most pages can stay as plain markdown files while still gaining richer rendering features from Nuxt Content, `remark-mdc`, and the docs theme.

## What You Can Use

- Standard Markdown for headings, links, lists, tables, and code blocks
- Raw HTML or Vue component tags inside markdown
- MDC syntax for Vue-like components, slots, props, and bindings
- `automd` blocks for generated content such as badges, file snippets, and API references

## Frontmatter

Use frontmatter when a page needs explicit metadata:

```yaml
---
title: My Page
icon: lucide:book-open
description: Short summary
---
```

## Standard Markdown And Components

Markdown pages can render built-in prose components and any local component you register in `.docs/components/`.

Examples:

- Alerts such as `::note` and `::warning`
- Package manager helpers such as `:pm-install` and `:pm-run`
- Local Vue components such as `<HeroBanner />`

```md
# Heading

Regular **Markdown** still works.

<HeroBanner />
```

## MDC Syntax

NuDocs supports the `remark-mdc` syntax used by Nuxt Content. In practice, these are the patterns you will use most often.

### Inline Components

Use a single `:` for inline components inside a paragraph:

```md
A simple :badge[beta]
A styled :badge[beta]{color="primary"}
How to say :hello{}-world in Markdown
```

### Block Components

Use `::` for components that wrap markdown content:

```md
::card
The content of the card.
::
```

If the component is alone on its own line, `:component` also works as sugar syntax for a block component.

### Named Slots

Block components can define named slots with `#slot-name`:

```md
::hero
Default slot content

#description
This goes into the `description` slot.
::
```

### Nested Components

Components can be nested by indenting their content. Adding more `:` makes the nesting easier to read:

```md
::hero
:::card
Nested content
:::
::
```

### Inline Props And Attributes

Use `{}` right after a component or markdown element to pass props and attributes:

```md
:badge[beta]{color="primary"}
[Nuxt](https://nuxt.com){target="\_blank"}
**Important**{.text-red-500}
```

Common shortcuts:

- `{#id}` for `id`
- `{.class-name}` for `class`
- `{disabled}` for boolean props
- `{:items='["a","b"]'}` for JSON-decoded arrays or objects

### YAML Props

When inline props get too long, switch to a YAML block:

```md
## ::feature-card

title: Fast
icon: lucide:zap
description: Build static docs with Nuxt Content.

---

::
```

### Variable Bindings

Frontmatter values can be referenced in markdown with `{{ }}`:

```md
---
packageName: nudocs
---

# {{ $doc.packageName || 'docs' }}
```

### Styled Spans

Use `[]` with inline props when you only need to style part of a sentence:

```md
Hello [World]{.font-bold .text-primary}!
```

## Generated Content

Use `automd` when content should be generated from code, metadata, or shared snippets:

```md
<!-- automd:file src="../.partials/warn.md" -->

> [!IMPORTANT]
> NuDocs is currently intended for NuJS docs only and is not fully customizable yet. <br>
> Contributions are more than welcome but please consider that this project is not ready yet to be used. <br>
> We don't guarantee stability yet and it is expected that it doesn't work time to time.

<!-- /automd -->
```

## Notes

- Standard Markdown remains the default. MDC is additive, not a replacement.
- Inline components placed alone on a line are treated like block components.
- Props can be passed to components and to native markdown elements such as links, images, code, italic, and bold text.
