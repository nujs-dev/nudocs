## Before You Build

Set the public site URL in `.config/docs.yaml` before running a production build:

```yaml [.config/docs.yaml]
url: "https://example.com"
```

This value is used for SEO metadata and generated files such as `/llms.txt`.

## Build

From your docs project root, run:

```bash
pnpm build
```

This runs `nudocs build` and generates a static site.

## Output Directories

NuDocs writes production build artifacts to `.output/` in the docs project root:

```text
docs/
  .output/
    public/
```

Use the directories as follows:

- `.output/` is the overall build output directory.
- `.output/public/` is the static site you publish to your hosting provider.

If your docs project lives in `docs/`, the full paths are `docs/.output/` and `docs/.output/public/`.

If your host asks for a publish directory, use `.output/public`.

## Static Assets

Files from `.docs/public/` are copied into the generated static site and served from the site root.

Example:

- `.docs/public/favicon.ico` becomes `/favicon.ico`
- `.docs/public/robots.txt` becomes `/robots.txt`

## Deploy To Static Hosting

NuDocs generates a static site, so the deployment flow is the same on most platforms:

1. Install dependencies.
2. Run `pnpm build`.
3. Publish `.output/public`.

Common platform settings:

- Cloudflare Pages: build command `pnpm build`, build output directory `.output/public`.
- Netlify: build command `pnpm build`, publish directory `.output/public`.
- Vercel: build command `pnpm build`, output directory `.output/public`.
- GitHub Pages: upload the contents of `.output/public` as the published site artifact.

## Notes

- `.output/` is generated and should not be edited manually.
- `.output/public/` is the directory most static hosts should serve.
- If you change your production domain, update `url` in `.config/docs.yaml` and rebuild.
